# WA Warmer

Портабл-приложение (Electron) для «прогрева» WhatsApp-аккаунтов со встроенным
движком [GOWA](https://github.com/aldinokemal/go-whatsapp-web-multidevice).
Аккаунты добавляются через QR-логин WhatsApp и автоматически переписываются
между собой (схема «все со всеми») со случайными интервалами; иногда
отправляются случайные картинки и ссылки из встроенного набора.

> ⚠️ Автоматизированная переписка нарушает условия использования WhatsApp и
> может привести к блокировке номеров. Используйте только свои аккаунты, на свой риск.

## Архитектура

- **GOWA** — headless-движок на Go (входит в комплект как бинарник). Подключается к
  WhatsApp напрямую (без браузера), держит несколько аккаунтов в одном процессе
  (scoping через `X-Device-Id`), отдаёт REST API на localhost.
- **Electron / main (Node)** — планировщик прогрева: выбирает пары, тайминги,
  контент; вызывает REST API GOWA.
- **Electron / renderer** — окно: аккаунты, QR, настройки, журнал.

Движок GOWA запускается как дочерний процесс на случайном свободном порту со
случайным basic-auth; состояние (SQLite сессий + `accounts.json`) хранится в папке
`data/` рядом с приложением — переносится вместе с портабл-сборкой.

## Разработка

```bash
npm install
npm start          # prestart сам скачает бинарник GOWA для вашей платформы и создаст контент-пак
```

Полезные скрипты:
- `npm run fetch-gowa` — скачать бинарники GOWA (по умолчанию все платформы; `-- --current` только текущую).
- `npm run seed-content` — перегенерировать встроенный контент-пак.

## Сборка портабл-версий

```bash
npm run build:mac   # dist/WA Warmer-<ver>-<arch>-mac.zip (x64 + arm64)
npm run build:win   # dist/WA Warmer-<ver>-portable.exe (один файл, без установки)
```

Перед сборкой конкретной платформы убедитесь, что скачаны нужные бинарники GOWA
(`npm run fetch-gowa`). Артефакты складываются в `dist/`.

### Автосборка (GitHub Actions)

Workflow [`.github/workflows/build.yml`](.github/workflows/build.yml) собирает оба
портабл-артефакта на нативных раннерах:

- **push тега `v*`** — собирает mac (x64+arm64) и Windows (x64) и **публикует их в GitHub Release**;
- **push в `main` / PR / ручной запуск** (`workflow_dispatch`) — собирает и складывает
  результаты в Artifacts прогона (без релиза).

Выпуск версии:

```bash
git tag v1.0.0 && git push origin v1.0.0
```

### Примечания по платформам
- **macOS**: сборка не подписана — при первом запуске: правый клик → «Открыть»,
  либо `xattr -dr com.apple.quarantine "WA Warmer.app"`.
- **Windows**: portable `.exe` — Windows SmartScreen может предупредить
  («Подробнее» → «Выполнить в любом случае»).

## Как идёт прогрев

Прогрев моделирует **живые диалоги**, а не разовые сообщения: планировщик выбирает
пару аккаунтов и разыгрывает между ними серию реплик туда-обратно с индикатором
«печатает…», отметками о прочтении и статусом «онлайн». Тексты варьируются
(вариативность `{a|b}`, эмодзи, без повторов подряд). Между диалогами — случайная
пауза, иногда более длинные перерывы; ночью и вне активных часов прогрев спит, по
выходным нагрузка ниже. Требуется **минимум 2 подключённых аккаунта**.

Настройки в окне:
- **Пауза между диалогами (мин, от/до)** — случайный интервал между разговорами.
- **Дневной лимит / аккаунт** — сколько исходящих сообщений отправит аккаунт в сутки.
- **Параллельных диалогов** — сколько разговоров идёт одновременно (масштабирование под
  много аккаунтов; аккаунт не участвует в двух диалогах сразу). Рассчитано на 20–50–100 аккаунтов.
- **Дней разгона (ramp-up)** — лимит растёт первые N дней у нового аккаунта.
- **Новый чат каждые N дней** — число разных собеседников растёт постепенно: день 1 —
  1 чат (аккаунты общаются парами), затем +1 собеседник каждые N дней. Связи постоянные.
- **Активность с/до (час)** — окно работы по локальному времени машины.
- **Картинки / Ссылки** — включать ли их в отправку.

По каждому аккаунту показывается: день прогрева, отправлено, принято. При отключении
аккаунта (logout/бан) приходит уведомление.

## Контент (data/content-pack/)

Контент **пользовательский** и лежит рядом с приложением — при первом запуске
создаётся папка `data/content-pack/`:
- `messages.txt` — тексты, по одному на строку (поддержка вариативности `{Привет|Хай}`);
- `links.txt` — ссылки, по одной на строку;
- `polls.txt` — опросы: `Вопрос? | вариант1 | вариант2 | вариант3`;
- `images/` — картинки (`.jpg/.png/.webp/...`);
- `voice/` — голосовые (`.ogg/.mp3/.m4a/...`);
- `stickers/` — стикеры (`.webp/.png`).

Файлы создаются пустыми — заполните своим контентом. В карточке «Контент» есть кнопки:
открыть папку, добавить картинки, обновить. Типы включаются галочками в настройках.
В переписке также используются реакции, цитаты и редкие «опечатки с правкой».

## Добавление аккаунтов

- **QR** — «+Добавить» → отсканировать код в WhatsApp → Связанные устройства.
- **По коду** — вкладка «По коду»: ввести номер, получить 8-значный код, ввести в телефоне.
- **Массово** — кнопка «Массово»: вставить список номеров → получить коды для каждого.

## Статистика

Кнопка **«📊 Статистика»** — окно с итогами (аккаунты/онлайн/отправлено/принято),
графиком отправлено/принято за 14 дней, полосами по аккаунтам и **экспортом в CSV**.
