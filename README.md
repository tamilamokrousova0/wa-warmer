# WA Warmer

Headless-сервер для «прогрева» WhatsApp-аккаунтов со встроенным движком
[GOWA](https://github.com/aldinokemal/go-whatsapp-web-multidevice) и веб-панелью
управления (браузер, без установки приложения). Аккаунты добавляются через
QR-логин WhatsApp и автоматически переписываются между собой (схема «все со
всеми») со случайными интервалами; иногда отправляются случайные картинки и
ссылки из встроенного набора.

> ⚠️ Автоматизированная переписка нарушает условия использования WhatsApp и
> может привести к блокировке номеров. Используйте только свои аккаунты, на свой риск.

## Архитектура

- **GOWA (пул движков)** — headless-движок на Go (входит в комплект как бинарник),
  подключается к WhatsApp напрямую (без браузера). Аккаунты разбиты на **4 фиксированные
  страновые группы** (Украина — основная, Германия/Польша/Англия — вспомогательные для
  разогрева UA-номеров), и **каждая группа запускает свой собственный процесс GOWA** на
  отдельном порту со своим basic-auth и своим SQLite — группы полностью изолированы
  (плохой прокси в одной группе не роняет остальные).
- **Node-ядро (`src/core`, `src/main`)** — планировщик прогрева: выбирает пары, тайминги,
  контент по языку пары; ходит в REST API нужного движка группы через `engineRouter`.
- **Веб-панель (`src/server`)** — Fastify-сервер: REST API + WebSocket-трансляция событий
  ядра, статика панели (`src/renderer`) отдаётся тем же процессом. Открывается в браузере
  (аккаунты, QR, настройки, журнал) — без Electron и без установки на клиентскую машину.

Каждый движок группы запускается как дочерний процесс на случайном свободном порту со
случайным basic-auth; состояние (SQLite сессий по группам + `accounts.json`) хранится в
папке данных (`WA_DATA_DIR`, по умолчанию `~/.wa-warmer/data`).

## Разработка

```bash
npm install
npm start          # prestart соберёт движок GOWA для вашей платформы и создаст контент-пак
```

Откройте `http://localhost:8760` — первый запуск попросит пароль панели
(`WA_PANEL_PASSWORD` в окружении).

> Для сборки движка нужен **Go ≥ 1.25** (macOS: `brew install go`). Движок собирается
> из исходников, потому что поддержка исходящего прокси (`--whatsapp-proxy`) есть
> только в ветке `main` GOWA и ещё не попала ни в один релиз. Скрипт закреплён на
> конкретном коммите и собирает с `-tags purego` (чистый Go SQLite, без cgo — это и
> даёт простую кросс-сборку).

Полезные скрипты:
- `npm run build-gowa` — собрать движок из исходников (по умолчанию текущая платформа; `-- --all` — mac-x64 + mac-arm64 + win-x64; `-- --target win-x64` — конкретная; `-- --current` — явно текущая).
- `npm run seed-messages` — перегенерировать встроенный контент-пак (`content-seed/`).

## Прокси (per-group, опционально)

У каждой из 4 групп — **свой** исходящий прокси (SOCKS5/HTTP/HTTPS) или прямое
соединение. Настраивается в панели (карточка «Группы») или через `POST /api/groups`:
локальный порт **9Proxy** (если ротация идёт через локальный SOCKS-прокси на машине) или
адрес резидентного прокси-провайдера — строкой вида `socks5://user:pass@host:1080`.
Кнопка «Проверить» показывает видимый IP для конкретного прокси. **Смена прокси у
группы перезапускает только движок этой группы** — остальные три продолжают работать без
разрыва сессий.

## Сервер на Mac mini (Tailscale)

Панель разворачивается как постоянный фоновый сервис на Mac mini (например, в домашней
сети), доступный только внутри вашей Tailscale-сети — без публикации в интернет.

```bash
git clone <repo> wa-warmer && cd wa-warmer
npm ci
npm run build-gowa -- --all   # нужен Go >= 1.25; --all соберёт mac-x64 + mac-arm64 + win-x64,
                               # для одного Mac mini достаточно --current
```

1. **Пароль панели** — задайте `WA_PANEL_PASSWORD` (используется только при самом первом
   запуске: хешируется и сохраняется в `<WA_DATA_DIR>/panel.json`, дальше переменную можно убрать).
2. **launchd-юнит** — скопируйте `deploy/com.wawarmer.panel.plist` в
   `~/Library/LaunchAgents/`, подставьте в нём реальные пути (`node`, репозиторий,
   `WA_DATA_DIR`) и пароль, затем:
   ```bash
   cp deploy/com.wawarmer.panel.plist ~/Library/LaunchAgents/com.wawarmer.panel.plist
   # отредактировать плейсхолдеры в скопированном файле
   launchctl load ~/Library/LaunchAgents/com.wawarmer.panel.plist
   ```
   `KeepAlive`/`RunAtLoad` держат сервер живым (автозапуск при загрузке, рестарт при падении).
3. **Доступ** — внутри Tailscale-сети откройте `http://<magicdns-имя-mac-mini>:8760`
   (MagicDNS-имя видно в `tailscale status` / консоли Tailscale).
4. **Опционально HTTPS** — `tailscale serve --bg 8760` поднимет HTTPS-фронт на
   `https://<magicdns-имя>.<tailnet>.ts.net` с автоматическим сертификатом, доступный
   только участникам той же tailnet.

Логи сервиса: `~/Library/Logs/wa-warmer/panel.log` (путь задан в plist).
Остановить/перезапустить: `launchctl unload` / `launchctl load` того же файла.

**Перенос со старой (Electron) установки.** Чтобы сохранить существующие сессии и
аккаунты, укажите `WA_DATA_DIR` на прежнюю папку данных — группа `ua` переиспользует её
`gowa/whatsapp.db` (ре-логин не нужен); иначе свежая папка данных стартует с нуля.

## Как идёт прогрев

Прогрев моделирует **живые диалоги**, а не разовые сообщения: планировщик выбирает
пару аккаунтов и разыгрывает между ними серию реплик туда-обратно с индикатором
«печатает…», отметками о прочтении и статусом «онлайн». Тексты варьируются
(вариативность `{a|b}`, эмодзи, без повторов подряд). Между диалогами — случайная
пауза, иногда более длинные перерывы; ночью и вне активных часов прогрев спит, по
выходным нагрузка ниже. Требуется **минимум 2 подключённых аккаунта**.

Настройки прогрева (в панели):
- **Пауза между диалогами (мин, от/до)** — случайный интервал между разговорами.
- **Дневной лимит / аккаунт** — сколько исходящих сообщений отправит аккаунт в сутки.
- **Параллельных диалогов** — сколько разговоров идёт одновременно (масштабирование под
  много аккаунтов; аккаунт не участвует в двух диалогах сразу). Рассчитано на 20–50–100 аккаунтов.
- **Дней разгона (ramp-up)** — лимит растёт первые N дней у нового аккаунта.
- **Новый чат каждые N дней** — число разных собеседников растёт постепенно: день 1 —
  1 чат (аккаунты общаются парами), затем +1 собеседник каждые N дней. Связи постоянные.
- **Активность с/до (час)** — окно работы по локальному времени машины.
- **Картинки / Ссылки** — включать ли их в отправку.

По каждому аккаунту показывается: день прогрева, отправлено, принято. При отключении
аккаунта (logout/бан) приходит уведомление.

## Контент (content-pack/)

Контент **пользовательский** и лежит в папке данных сервера (`WA_DATA_DIR`) — при
первом запуске создаётся `content-pack/`. Тексты и ссылки разбиты по группам-странам,
а медиа общие для всех групп:
- `content-pack/<ua|de|pl|uk>/messages.txt` — тексты для группы, по одному на строку
  (поддержка вариативности `{Привет|Хай}`);
- `content-pack/<ua|de|pl|uk>/links.txt` — ссылки группы, по одной на строку;
- `content-pack/shared/images/` — картинки (`.jpg/.png/.webp/...`), общие для всех групп;
- `content-pack/shared/voice/` — голосовые (`.ogg/.mp3/.m4a/...`), общие для всех групп.

Файлы создаются пустыми — заполните своим контентом. В карточке «Контент» есть кнопки:
открыть папку, добавить картинки, обновить. Типы включаются галочками в настройках.
В переписке также используются реакции, цитаты и редкие «опечатки с правкой».

## Добавление аккаунтов

- **QR** — «+Добавить» → отсканировать код в WhatsApp → Связанные устройства.
- **По коду** — вкладка «По коду»: ввести номер, получить 8-значный код, ввести в телефоне.
- **Массово** — кнопка «Массово»: вставить список номеров → получить коды для каждого.

## Статистика

Кнопка **«📊 Статистика»** — раздел с итогами (аккаунты/онлайн/отправлено/принято),
графиком отправлено/принято за 14 дней, полосами по аккаунтам и **экспортом в CSV**.
